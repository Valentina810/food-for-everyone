1. Регистрация пользователя:
- Пользователь открывает приложение и выбирает опцию "Регистрация".
- Пользователь вводит свои персональные данные, такие как имя, фамилия, адрес, контактная информация.
- Пользователь выбирает свой статус, например, "малоимущий" или "бездомный".
- Пользователь создает учетную запись с помощью уникального логина и пароля.

2. Изменение данных пользователя:
- Пользователь открывает приложение и выбирает опцию "Мой профиль".
- Пользователь вносит изменения в свои персональные данные, такие как адрес или контактная информация.
- Пользователь сохраняет изменения.

3. Заказ еды:
- Пользователь открывает приложение и выбирает опцию "Заказать еду".
- Пользователь просматривает доступное меню и выбирает понравившиеся блюда.
- Пользователь указывает количество порций каждого блюда.
- Пользователь подтверждает заказ.

4. Отмена заказа:
- Пользователь открывает приложение и выбирает опцию "Мои заказы".
- Пользователь выбирает заказ, который хочет отменить.
- Пользователь подтверждает отмену заказа.

5. Пожертвование за питание:
- Пользователь открывает приложение и выбирает опцию "Сделать пожертвование".
- Пользователь указывает сумму, которую он хочет пожертвовать.
- Пользователь выбирает способ платежа и подтверждает пожертвование.

6. Подтверждение заказа (для администратора):
- Администратор открывает приложение и выбирает опцию "Подтвердить заказ".
- Администратор просматривает список заказов, которые ожидают подтверждения.
- Администратор выбирает заказ и подтверждает его выполнение.

7. Отклонение заказа (для администратора):
- Администратор открывает приложение и выбирает опцию "Отклонить заказ".
- Администратор просматривает список заказов, которые ожидают подтверждения.
- Администратор выбирает заказ и отклоняет его, указав причину отклонения.

8. Добавление пользователя (для администратора):
- Администратор открывает приложение и выбирает опцию "Добавить пользователя".
- Администратор вводит персональные данные нового пользователя, такие как имя, фамилия, адрес, контактная информация.
- Администратор указывает статус пользователя, например, "малоимущий" или "бездомный".
- Администратор создает учетную запись для нового пользователя с помощью уникального логина и пароля.

9. Удаление пользователя (для администратора):
- Администратор открывает приложение и выбирает опцию "Удалить пользователя".
- Администратор выбирает пользователя, которого он хочет удалить.
- Администратор подтверждает удаление пользователя.

10. Редактирование данных пользователя (для администратора):
- Администратор открывает приложение и выбирает опцию "Редактировать данные пользователя".
- Администратор выбирает пользователя, данные которого он хочет изменить.
- Администратор вносит изменения в персональные данные пользователя, такие как адрес или контактная информация.
- Администратор сохраняет изменения.

11. Блокировка пользователя (для администратора):
- Администратор открывает приложение и выбирает опцию "Заблокировать пользователя".
- Администратор выбирает пользователя, которого он хочет заблокировать.
- Администратор указывает причину блокировки пользователя.
- Пользователь больше не может заказывать еду через приложение.
